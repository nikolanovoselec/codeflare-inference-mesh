# Observability

This domain covers response metadata, admin status, node metrics, audit events, and failure reporting.

---

### REQ-OBS-001: Provider response metadata

**Intent:** Clients and operators need request identifiers and routing metadata to debug which node served a request without exposing private node topology unnecessarily.

**Applies To:** Client

**Acceptance Criteria:**

1. Provider responses include a request ID header generated by the router.
2. Provider responses include the selected public session ID when one was accepted or generated.
3. Provider responses include the selected node display name or stable node ID when policy permits disclosure.
4. Error responses include the request ID in either headers or the response body.
5. Response metadata never includes bearer tokens or Cloudflare API credentials.

**Constraints:** [CON-SEC-001](constraints.md#con-sec-001-separate-credential-classes), [CON-SCHED-001](constraints.md#con-sched-001-serialized-live-reservations)

**Priority:** P1

**Dependencies:** [REQ-RTR-002](router-worker.md#req-rtr-002-chat-completion-forwarding)

**Verification:** Automated test

**Status:** Planned

### REQ-OBS-002: Admin status surface

**Intent:** Admins need a current fleet view to see node readiness, session placement, active profiles, and routing capacity.

**Applies To:** Admin

**Acceptance Criteria:**

1. Admin status lists registered nodes with status, public models, active profiles, in-flight count, capacity, and last-seen age.
2. Admin status lists active session mappings with session ID, node ID, public model, profile ID, and upstream model name.
3. Admin status includes lease expiration and failure penalty state for each node.
4. Admin status omits token hashes and plaintext credentials.
5. Admin status returns a machine-readable JSON shape for UI and automation use.

**Constraints:** [CON-STATE-001](constraints.md#con-state-001-d1-is-durable-truth), [CON-SEC-002](constraints.md#con-sec-002-no-plaintext-durable-secrets)

**Priority:** P1

**Dependencies:** [REQ-SCH-003](state-scheduling.md#req-sch-003-node-eligibility-and-busy-response), [REQ-SCH-004](state-scheduling.md#req-sch-004-session-affinity)

**Verification:** Automated test

**Status:** Planned

### REQ-OBS-003: Node metrics

**Intent:** The router should make scheduling decisions and operator screens from concrete node health and performance signals, not just heartbeat presence.

**Applies To:** Node Agent

**Acceptance Criteria:**

1. Heartbeat metrics include runtime state, loaded profile, in-flight requests, and last request duration.
2. Heartbeat metrics include prompt throughput and generation throughput when the runtime reports them.
3. GPU metrics are best-effort from platform tools first and absent rather than fabricated when unavailable.
4. Metrics include WARP status and Mesh IP detection result.
5. The scheduler can apply recent failure and health signals when scoring nodes.

**Constraints:** [CON-SCHED-001](constraints.md#con-sched-001-serialized-live-reservations), [CON-RUNTIME-001](constraints.md#con-runtime-001-llamacpp-first-runtime)

**Priority:** P1

**Dependencies:** [REQ-NODE-002](node-agent.md#req-node-002-node-claim-and-heartbeat), [REQ-RUN-003](runtime-profiles.md#req-run-003-managed-llamacpp-runtime)

**Verification:** Automated test

**Status:** Planned

### REQ-OBS-004: Failure reporting

**Intent:** Normal operational failures should produce actionable state transitions and errors rather than ambiguous `500` responses.

**Applies To:** Admin

**Acceptance Criteria:**

1. Missed heartbeats expire a node lease and remove the node from eligible scheduling.
2. Mid-stream node failures release reservations and increase the node's recent failure score.
3. WARP disconnect reports make the node ineligible until a healthy heartbeat returns.
4. Capacity exhaustion returns busy state rather than internal error state.
5. Audit events record setup, claim, revoke, route provisioning, profile switch, and deployment actions.

**Constraints:** [CON-SCHED-001](constraints.md#con-sched-001-serialized-live-reservations), [CON-STATE-001](constraints.md#con-state-001-d1-is-durable-truth)

**Priority:** P1

**Dependencies:** [REQ-SCH-003](state-scheduling.md#req-sch-003-node-eligibility-and-busy-response), [REQ-RTR-003](router-worker.md#req-rtr-003-streaming-pass-through)

**Verification:** Automated test

**Status:** Planned
